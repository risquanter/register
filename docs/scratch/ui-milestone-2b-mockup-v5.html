  • 1Three-state control: Off = today's single-branch chart. Overlay = one chart, all selected branches; colour becomes branch identity, so per-node distinction must move to shades within each branch's palette — readable to roughly 3–4 branches × 1–3 nodes. Side by side = one panel per branch on shared pinned axes; node colours keep their normal meaning inside each panel, scaling to more branches by tiling — panels keep a minimum width and wrap to further rows automatically. Diff panel is pairwise in every state — with 3+ branches the diff pair is picked explicitly.
  • 2The ✎ markers show input edits between two chosen branches (hash diff, UC5): what the user changed, not which curves differ — in a with/without-mitigation comparison every curve differs while the edit list is exactly the mitigation change. The separate diff list under the chart was removed as clutter; the edit markers in the tree carry the same information in place.

Proposes: right panel of Analyze gains a History tab (commit messages are per-tree and human-readable since DD-7). The fork action is the single sanctioned write affordance in Analyze — it creates a new branch and hands off to Design.
