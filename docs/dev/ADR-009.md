# ADR-009: Compositional Risk Aggregation via the TrialOutcomes Monoid

**Status:** Accepted  
**Date:** 2026-07-17 (supersedes the 2026-01-17 revision, which placed the monoid on `RiskResult`)  
**Tags:** architecture, domain-model, zio-prelude, aggregation, monoid

---

## Context

- Aggregate nodes must produce loss distributions for LEC visualisation even though only leaf nodes carry distribution parameters â€” consumers need one read interface for both kinds
- Trial-aligned outcomes combine by pointwise summation, but a combine is only a lawful monoid on data that excludes node identity: a parent's node ID has no principled derivation from its children's IDs, so any instance on an identity-carrying type must invent one
- Parallel and order-independent aggregation is safe only when licensed by stated, tested algebraic laws â€” not by convention
- The identity element's `nTrials` derives from configuration, so an `Identity` instance must be scoped to `SimulationConfig` to avoid trial-count drift
- Every node's LEC curve must be traceable to the contributing leaves (ADR-003); provenance belongs to the node that was simulated, never to the aggregate

---

## Decision

### 1. The algebra lives on `TrialOutcomes`, not on a result type

`LossDistribution` is the product `NodeId Ă— TrialOutcomes`: the node ID is a label supplied by tree context; only the trial data participates in combination.

```scala
case class TrialOutcomes(nTrials: Int, outcomes: Map[TrialId, Loss])

object TrialOutcomes {
  def empty(using cfg: SimulationConfig): TrialOutcomes           // identity element
  def combine(a: TrialOutcomes, b: TrialOutcomes): TrialOutcomes  // outer-join pointwise sum,
                                                                  // same-nTrials alignment enforced
  given commutative: Commutative[TrialOutcomes]                   // extends Associative â€” one instance, both laws
  given identity(using SimulationConfig): Identity[TrialOutcomes] // config-scoped, opt-in
}
```

### 2. Uniform read interface, distinct constructors

Consumers accept the sealed supertype; the subtype distinguishes a simulated leaf from an aggregate:

```scala
sealed abstract class LossDistribution(val nodeId: NodeId, val trialOutcomes: TrialOutcomes) extends LECCurve

case class RiskResult(nodeId, trialOutcomes, provenances)   // leaf: Monte Carlo sampled
case class RiskResultGroup(children, nodeId, trialOutcomes) // portfolio: aggregated, children retained
```

`LECGenerator`, the resolver, and the query knowledge base all take `LossDistribution`.

### 3. Portfolio construction is a named constructor, not a monoid reduction

The parent ID comes from the tree; the outcomes come from the algebra:

```scala
// RiskResultResolverLive â€” portfolio branch
RiskResultGroup(portfolio.id, childResults*)   // childResults: List[LossDistribution]
// internally: children's TrialOutcomes reduced with TrialOutcomes.combine
```

### 4. Laws license parallel child resolution

Associativity and commutativity make evaluation order irrelevant to the figures, so the resolver resolves portfolio children concurrently:

```scala
childResults <- ZIO.foreachPar(portfolio.childIds.toList)(resolveChild)
// foreachPar preserves list order â†’ provenance order identical to sequential
```

### 5. Provenance is read through the structure, never merged into the aggregate

```scala
// RiskResult: carries its own content-only records (DD-19: no riskId inside)
// RiskResultGroup: exposes NO flat provenance list â€” attribution is structural:
group.children.collect { case r: RiskResult => r.nodeId -> r.provenances }
```

A portfolio's provenance is read by walking its children at access time, pairing each
child's `nodeId` with its records one level above any flattening (DD-19, 2026-07-18:
the unattributed flat list on the supertype was removed together with
`NodeProvenance.riskId`).

## Code Smells

### âťŚ Monoid instance on an identity-carrying type

```scala
// BAD: combine has no principled nodeId for the result â€” it must steal one
given Associative[RiskResult] with
  def combine(a: => RiskResult, b: => RiskResult) = RiskResult(a.nodeId, ...)

// GOOD: instance on the identity-free payload
given commutative: Commutative[TrialOutcomes]
```

### âťŚ Aggregate typed as a leaf

```scala
// BAD: portfolio result indistinguishable from a simulated leaf; children lost
childResults.reduce(combine).withNodeId(portfolio.id)

// GOOD: distinct aggregate type, children retained for drill-down
RiskResultGroup(portfolio.id, childResults*)
```

### âťŚ Manual trial alignment

```scala
// BAD: hand-rolled per-trial summation at the call site
allTrials.map(t => t -> children.map(_.outcomes.getOrElse(t, 0L)).sum).toMap

// GOOD: the single combine implementation
children.map(_.trialOutcomes).reduceOption(TrialOutcomes.combine)
```

### âťŚ Unlicensed parallelism

```scala
// BAD: foreachPar over an aggregation with no stated associativity law
// GOOD: laws stated as zio-prelude instances, verified by property tests
//       (TrialOutcomesSpec), then foreachPar
```
